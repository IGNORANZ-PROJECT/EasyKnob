大切な注意事項

EasyKnobを使う前に、プライバシーと音量に関する注意を確認してください。

  • マイク音声はサーバーへ送信しません。音声処理はブラウザ内で行われます。
  • マイク権限が必要です。ONを押すと、ブラウザからマイク使用許可を求められます。
  • 音量に注意してください。ヘッドホンやスピーカーの音量を低めにしてから起動してください。
  • 外部アプリ側の設定も影響します。Discord、OBS、VRChat、VB-CABLE、BlackHoleなどの設定によって、音質や遅延が変わります。
  • 利用は自己責任です。聴覚、機器、配信、通話、OSやブラウザ設定に関する問題について、作者およびIGNORANZ PROJECTは責任を負いません。

この注意事項は初回のみ表示されます。詳しい使い方は「初めての方へ」とREADMEを確認してください。

設定

初めての方へ

EasyKnobでできること

EasyKnobは、マイクの声にECHOやREVERBなどをかけて、Discord、VRChat、OBS、通話アプリなどへ渡すためのブラウザアプリです。音声処理はブラウザ内で行い、マイク音声をサーバーへ送信しません。

まずはInputでマイク、Outputで出力先を選び、ONを押します。ブラウザのマイク許可が出たら許可してください。

基本の使い方

  1. Input で使うマイクを選びます。迷ったらDefaultのままで構いません。
  2. Output で音を出す先を選びます。DiscordやVRChatに渡す場合はVB-CABLEやBlackHoleを選びます。
  3. Preset で用途に近い音を選びます。歌ならSing、会話ならTalkが目安です。
  4. ON を押して起動します。状態がLIVEになれば動作中です。
  5. ノブを回して音を調整します。ON/OFF表示を押すと、そのノブだけ一時的に無効化できます。

外部アプリへ渡す時に必要なもの

EasyKnob本体はブラウザだけで動きます。ただしDiscordやVRChatに加工後の声を渡すには、仮想オーディオデバイスが必要です。

  • Windows: VB-CABLE
  • Mac: BlackHole
  • 推奨ブラウザ: Chrome / Edge

Windows / VB-CABLE

  1. VB-CABLEをインストールします。
  2. EasyKnobのOutputを CABLE Input にします。
  3. Discordなどの入力を CABLE Output にします。
  4. Discord側のマイクテストやノイズ抑制をOFFにして、遅延や音質を確認します。

Mac / BlackHole

  1. BlackHoleをインストールします。
  2. EasyKnobのOutputを BlackHole にします。
  3. Discordなどの入力を BlackHole にします。
  4. 必要に応じてAudio MIDI設定でBlackHoleの入出力を確認します。

ノブの意味

  • MIC: マイク入力の大きさです。0にするとEasyKnobから出る声も0になります。
  • ECHO: 後ろで小さく返ってくる遅れ音です。
  • REVERB: 声の後ろに響きを足します。REVERBカードの詳細ボタンからFREQ、GAIN、Qを調整できます。
  • ROOM: 空間の広さや左右の広がりです。
  • WET: エフェクト音の混ざり具合です。下げるほど原音に近くなります。
  • TONE: 声の明るさです。
  • AIR: 高域の抜けや空気感です。
  • STABLE: 音量差や刺さる部分を抑えて声を安定させます。
  • DOUBLE: 声を少し重ねて厚みを足します。

さらに詳しい説明は、設定画面の各Knob(ノブ)の項目にも記載しています。

低遅延・音質

音質を優先する場合は、Discordや配信ソフト側のノイズ抑制、エコー除去、自動ゲイン調整をOFFにしてください。Windows側の音声拡張、空間オーディオ、メーカー製マイク補正もOFF推奨です。

遅延が気になる場合は、Windowsの「このデバイスを聴く」、Discordのマイクテスト、OBSの音声モニタリングなど、別経路の監視音をOFFにしてください。

EasyKnob側では、不要なSTABLE、ECHO、REVERB、ROOM、WET、AIR、DOUBLEをOFFまたは低めにすると原音に近くなります。AUDIO ANALYZERも不要な時はOFF推奨です。

各Knob(ノブ)の役割は、設定画面のノブ一覧に記載しています。

ハウリング対策

スピーカーから出した音をマイクが拾うとハウリングします。基本はヘッドホンか仮想オーディオ出力を使い、スピーカー監視は避けてください。HOWLが出る時は、MIC、WET、ECHO、REVERB、DOUBLEを下げ、Outputをスピーカー以外にしてください。

音が出ない時

  • ブラウザのマイク許可がブロックされていないか確認してください。
  • Outputと、DiscordやVRChat側の入力デバイスの組み合わせを確認してください。
  • Windowsの場合、Discord側はCABLE Output、EasyKnob側はCABLE Inputです。
  • Macの場合、EasyKnob側も外部アプリ側もBlackHoleを選ぶ構成が基本です。
  • 別アプリのマイクテストやモニタリングを同時に使うと、遅延や二重音の原因になります。

対応外環境について

Safariや一部ブラウザでは、出力先デバイスの選択が使えない場合があります。対応外環境では、EasyKnobを仮想マイク用途に使えない可能性があります。
